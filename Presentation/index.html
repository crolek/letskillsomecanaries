The Canary Test

Originated in Coal Mines

Image from: http://www.academia.dk/Blog/wp-content/uploads/CanaryInACoalMine_2.jpg

Canary dies to carbon monoxide before the miner.

The 'rough' definition: “A test that you run to make sure that everything is setup and ready for development”

  • You’ll find missing dependencies (mvn/npm install)
  • SDK or Path issues
  • It also makes sure you’re in the right mindset about the context of what you’re working on.

Diving into a new app..

  1. Get it stood up (find an entry point)
    Is your Canary alive?
  2. Make the simplest of changes
    What's your feedback cycle like?
  3. Then start looking at the problem
    You're in a workable state
  4. Why step 2? because maybe you’re working on the wrong template, or you’re not working something you expect.
